MHF Project

Modern Honeynet Framework Project

This project has been a long time coming. I first started thinking about honeypots when my friend (and security expert) Adam Armstrong introduced me to them in 2012. I started to realize honeypots were becoming more popular in the corporate world when I read this article, which mentions successful security companies like TopSpin Security. They certainly aren't the only ones using virtual machines to lure in hackers and then analyze what they do.
When I attended the OWASP AppSec Conference 2015 in Santa Monica in January and was talking to a friend about hackers, it became clear I really don't know what a hacker does once they've gained root level priviledges. Do they cover their tracks? Do they ensure they have a way back into the system? Do they try to attack other computers on the networks? I really did not know.
A honeynet is a great way to answer some of these questions. So, using AWS EC2 I launched some servers that were vulnerable and waited. I also setup some vulnerable server using the Modern Honeynet Framework, a project that was created by Jason Trost. I am still in the process of analyzing data, but I will update this page with more information as I discover it.
